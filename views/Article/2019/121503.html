                    # 1.3 前端与后端概念分析

                    一般网站类项目都有前台和后台之分。

                    • 前台: 给网友（游客）使用页面叫做前台。
                    • 后台: 系统管理员使用页面

                    # 1.3 需求分析

                    在开发任何一个功能的时候，都需要前端程序员和后端程序员共同来完成。

                    • 前端程序员: 写 css、html、js的程序员，主要任务是页面显示结构、样式，页面特效，==Ajax==
                    • 后端程序员: 写php、jsp、asp、nodejs的程序员，主要任务是对数据表进行增删改查操作

                    因为是模拟所以就是一人多角色，没有后端程序员，所以我们自己得充当两个角色。 通过页面能够基本掌握项目的功能和实现目标

                    • 前台：
                      • 首页: 信息聚合页面，各种方式显示文章列表
                      • 列表页: 显示某个栏目下的所有文章列表
                      • 详情页: 显示文章的详细内容
                    • 后台：
                      • 管理员登录功能
                      • 文章管理：文章的增删改查
                      • 栏目管理：栏目的增删改查
                      • 用户管理：用户的增删改
                      • 评论管理：批准/驳回评论
                      • 设置：图片轮播、网站设置等。

                    # 2.项目结构搭建

                    在实现上述描述的各项功能之前，必须先定好在哪里开发这个项目，项目所需的包有哪些，项目的目录结构是怎样的才够清晰，等等一系列问题，这是做项目之前都要解决的。

                    在自己的计算机上找一个位置，创建一个文件夹，比如 alishow。然后我们将项目代码全部放到这里面。

                    # 2.1 安装项目所需的包

                    分析项目所需的包：

                    • express：搭建服务必须的包
                    • art-template：模板引擎
                    • express-art-template：专门配合express使用的模板引擎
                    • express-session：session管理（用户登录后，需要使用session记录用户的状态）
                    • mysql：对MySQL数据库操作，必须用到的包
                    • moment：时间处理包（比如登录时间、文章发布时间等地方可能会需要到）
                    • multer：文件上传需要用的包（会员头像上传、文章或栏目配图都需要）

                    在alishow目录中，执行下面的命令，安装项目所需的包

                    npm init -y
npm i express art-template express-art-template express-session mysql multer moment body-parser

                    # 2.5 路由文件命名规则

                    阿里百秀项目会涉及到非常非常多的路由（接口地址），所以不能把路由全部定义到app.js中。应该按功能将路由定义到不同的文件中，我们把这些路由文件全部放到router文件夹中，显得文件比较整齐。

                    关键问题是如何给路由文件命名？当然定义成 a.js 或 b.js 之类的名字也可以，但是这样的名字没有意义，而且路由文件太多的话，也分不清谁是谁了。

                    ==当前项目，可以按模板文件夹来定义路由，也就是路由文件的名字和模板所在的文件夹同名，这样做的好处是，看到路由文件的名字就知道它里面存放了哪些路由。==

                    • 比如 home.js 对应 home 文件夹，前台所有相关的路由都放到这里面
                    • 比如 admin.js 对应 admin 文件夹， 和admin文件夹中的 index.html、login.html相关的路由都放到这里面
                    • 比如 admin_cate.js 对应 admin/cate文件夹， 该路由处理admin/cate文件夹中的文件相关的路由
                    • ....

                    下面是定义路由文件的示意：

                    router
  - home.js       ---  处理前台各个页面
  - admin.js      ---  处理后台admin文件夹中的文件的 及 和这些文件相关的路由
  - admin_cate.js --- 处理admin/cate文件夹中的文件的 及 和这些文件相关的路由
  - admin_xxx.js  --- 处理admin/xxx文件夹中的文件的 及 和这些文件相关的路由
  - ....js

                    # 2.6 定义路由

                    路由文件有了，那么路由如何来定义呢？其实将路由定义的和文件所在路径一样就可以了

                    比如 :

                    前台路由
 /                             - 表示访问前台首页index.html
 /list.html                    - 表示访问前台list.html
 /detail.html                  - 表示访问前台的detail.html
后台路由
 /admin/index.html             - 表示访问后台首页 view/admin/index.html
 /admin/login.html             - 表示访问后台登录页面 view/admin/login.html
 /admin/cate/addcate.html      - 表示访问 view/admin/cate/addcate.html

                    # 2.8 抽取页面公共部分

                    <html>
